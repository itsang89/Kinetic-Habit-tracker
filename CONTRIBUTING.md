# Contributing to Kinetic

Thank you for your interest in contributing to Kinetic! We want to make contributing to this project as easy and transparent as possible.

## Code of Conduct

By participating in this project, you agree to abide by the terms of our Code of Conduct. (Link to your CoC or add a basic one here).

## Our Development Process

1.  **Fork the repository** and create your branch from `main`.
2.  **Install dependencies** using `npm install`.
3.  **Run the development server** with `npm run dev`.
4.  **Make your changes**.
5.  **Lint your code** using `npm run lint`.
6.  **Issue a Pull Request**!

## Coding Style

- Use **TypeScript** for all new code.
- Follow the existing monochrome design system (Black/White/Gray).
- Use **Tailwind CSS 4** for styling.
- Ensure animations are smooth and use **Framer Motion**.
- Add comments for complex logic (especially in `src/store` or `src/lib`).

## Pull Request Guidelines

- Ensure your code builds and passes linting.
- Provide a clear description of the changes.
- Attach screenshots or screen recordings for UI changes.
- Update documentation (like this file or `ARCHITECTURE.md`) if necessary.

## Questions?

If you have any questions, feel free to open an issue!
